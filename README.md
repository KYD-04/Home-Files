# HomeFiles - Файловый сервер для локальной сети

## Описание

**HomeFiles** - это лёгкий инструмент для быстрого шаринга файлов устройства по локальной сети. Проект представляет собой простой файловый сервер, разработанный для удобного обмена файлами между устройствами в домашней или офисной сети.

### Основные возможности

- **Управление файлами**: Добавление и удаление файлов/папок в общий доступ через веб-интерфейс
- **Двойной интерфейс**: Административный (порт 8110) и публичный (порт 8111) интерфейсы
- **Отзывчивый дизайн**: Современный Material Design интерфейс с поддержкой мобильных устройств
- **Скачивание файлов**: Прямое скачивание файлов и создание ZIP архивов для папок
- ⬆**Загрузка файлов**: Возможность загрузки файлов от внешних пользователей
- **Проверка статуса**: Автоматическая проверка существования файлов и их размеров
- **Гибкая конфигурация**: Настройка через YAML файл с поддержкой множества типов файлов
- **Безопасность**: CORS поддержка и базовая валидация файлов

### Поддерживаемые форматы файлов

**Документы**: txt, pdf, doc, docx, rtf, odt, md, csv, xlsx, xls, pptx, ppt

**Изображения**: jpg, jpeg, png, gif, bmp, svg, webp

**Аудио**: mp3, wav, flac, ogg

**Видео**: mp4, avi, mkv, mov, wmv

**Архивы**: zip, rar, 7z, tar, gz

**Код**: py, js, html, css, json, xml, yaml, yml, java, cpp, c, cs, php, rb, go, rs

## Установка и запуск

### Требования

- Python 3.12+
- pip

### Установка

1. Клонируйте репозиторий или скачайте файлы проекта:
```bash
git clone https://github.com/KYD-04/Home-Files.git
cd HomeFiles
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Настройте конфигурацию (опционально):
```bash
# Отредактируйте config.yaml под ваши нужды
nano config.yaml
```

### Запуск

```bash
python run.py
```

После запуска сервер будет доступен по адресам:
- **Административный интерфейс**: http://127.0.0.1:8110
- **Публичный интерфейс**: http://[ваш-ip]:8111

## Использование

### Административный интерфейс (порт 8110)

1. Откройте http://127.0.0.1:8110 в браузере
2. Нажмите "Добавить файл" для добавления файлов/папок в общий доступ
3. Используйте "Обновить" для синхронизации списка файлов
4. Управляйте общими файлами и папками

### Публичный интерфейс (порт 8111)

1. Откройте http://[ваш-ip]:8111 в браузере
2. Просматривайте доступные файлы и папки
3. Скачивайте файлы или создавайте ZIP архивы папок
4. Загружайте новые файлы (если разрешено)

## Конфигурация

Основные настройки находятся в файле `config.yaml`:

```yaml
# Путь для загрузок
upload_path: "./uploads"

# Максимальный размер файла
max_file_size: "100GB"

# Разрешенные расширения
allowed_extensions:
  - "txt"
  - "pdf"
  - "jpg"
  # ... и другие

# Настройки сети
network:
  bind_address: "0.0.0.0"
  admin_port: 8110
  public_port: 8111
```

## API Endpoints

### Административные (порт 8110)
- `GET /` - Главная страница администрирования
- `GET /api/files` - Получение списка файлов
- `POST /api/add-file` - Добавление файла
- `DELETE /api/remove-file/<id>` - Удаление файла
- `GET /download/<id>` - Скачивание файла
- `GET /download-folder/<id>` - Скачивание папки как ZIP
- `POST /upload` - Загрузка файла
- `POST /shutdown` - Завершение работы сервера

### Публичные (порт 8111)
- `GET /` - Публичная страница
- `GET /api/files` - Получение списка доступных файлов
- `GET /download/<id>` - Скачивание файла
- `GET /download-folder/<id>` - Скачивание папки как ZIP
- `POST /upload` - Загрузка файла

## Безопасность

- Сервер предназначен для использования в локальных сетях
- Не рекомендуется открывать публичный доступ к административному интерфейсу
- Поддерживается базовая валидация типов файлов
- CORS заголовки настроены для локального использования
